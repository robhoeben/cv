Html & CSS

Er wordt vaak gezegd, "CSS is geen echte programmeertaal". En dat klopt, maar dat betekent niet dat het daardoor eenvoudig is. Het is juist heel lastig. Dat komt door de eenvoud van de taal, en alle restricties die eraan verbonden zijn.
De beste uitspraak die ik al heb gehoord is:

"CSS is easy to learn
but difficult to master"

Wat mij betreft is dit de perfecte beschrijving van CSS. Bij het schrijven van HTML en CSS komt zoveel meer kijken dan vaak wordt gedacht; je schrijft abstracte regels waarmee onbekende content wordt georganiseerd in een onbekend medium.
Dat is erg lastig.

CSS geeft ook geen errors dus je weet dus niet zomaar wanneer iets kapot is. En een styling issue is niet simpel te debuggen. Websites moeten er op alle devices goed uitzien, maar browsers houden zich niet allemaal aan de standaarden, en hebben allemaal hun eigen quirks.

Ik heb al zóveel CSS geschreven dat ik zo ongeveer alle bugs wel een keer heb zien langskomen. Dit heeft er voor gezorgd dat ik de meeste bugs heel snel kan tackelen, of beter nog, kan voorkomen.

RWD

Met alle verschillende devices die tegenwoordig op het web surfen is ook Responsive webdesign een belangrijke techniek geworden. Een website moet op alle apparaten goed functioneren en op diverse schermgroottes worden weergegeven. Meer dan 50% van de websites wordt bezocht op mobiele devices, dus het dus geen overweging meer om voor RWD te kiezen, maar een must. Vanuit de 'mobile-first' gedachte zet ik een website vanuit een ontwerp naadloos om naar HTML & CSS.
